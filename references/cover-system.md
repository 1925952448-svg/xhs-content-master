# 封面系统 · 参数与设计规范

> 配套脚本 `scripts/render_cover.py`，模板 `assets/cover_template.html`，配色 `assets/themes.json`。

## 为什么要分主题配色

小红书信息流是**白底 + 明亮生活图**的环境。深色、小字、千篇一律的封面会「沉下去」、看不清、且容易被算法判为同质化低质。
本系统用**高对比大字 + 关键词高亮 + 按内容类型分色**，让封面在 feed 里跳出来。

## 主题（`-t` 或 YAML `theme`）

| 值 | 名称 | 适用内容类型 | 主色调 |
|---|---|---|---|
| `blue` | 知识蓝 | 知识科普、教程、清单 | 深蓝 + 黄高亮 |
| `green` | 干货绿 | 实操步骤、方法、避坑 | 墨绿 + 黄高亮 |
| `red` | 观点红 | 观点、争议、内幕 | 暗红 + 黄高亮 |
| `light` | 清新白 | 生活、情感、轻松向 | 浅白 + 蓝高亮 |

> 同一账号建议**用颜色区分内容支柱**（如知识=蓝、干货=绿、观点=红），形成可识别的视觉体系。

## YAML 字段

| 字段 | 必填 | 说明 |
|---|---|---|
| `theme` | 否 | 主题，默认 `blue`，命令行 `-t` 可覆盖 |
| `tag` | 建议 | 左上角胶囊标签（栏目名/系列名） |
| `eyebrow` | 否 | 主标题上方的小一号引导行（如「房间总是乱？」） |
| `title` | 是 | 主标题。`\n` 换行；`**关键词**` 自动高亮 |
| `subtitle` | 否 | 副标题。`\n` 换行；`**文字**` 加粗 |
| `chips` | 否 | 底部关键词数组，建议 2-3 个 |
| `author` | 否 | 署名昵称 |
| `author_sub` | 否 | 署名副标题（一句话简介） |
| `avatar` | 否 | 头像内 1 个字，默认取 `author` 首字 |

## 文案排版建议

- **主标题 ≤ 12 字/行，最多 2-3 行**：太长在缩略图看不清。
- **每张封面只高亮 1 个核心词**（数字、结果、冲突词），用 `**...**` 包裹。
- **副标题给「利益点」**：告诉用户点进去能得到什么。
- **数字优先**：「3 本」「5 个」「90%」比抽象描述更抓眼。

## 命令示例

```bash
# 用 YAML 里的 theme
python scripts/render_cover.py demos/sample_blue.md

# 命令行覆盖主题、指定输出
python scripts/render_cover.py note.md -t green -o output/cover.png
```

## 自定义

- 改配色：编辑 `assets/themes.json`，可新增主题键。
- 改版式：编辑 `assets/cover_template.html`（`{{占位符}}` 由脚本注入）。
- 换浏览器：设环境变量 `CHROME_PATH` 指向 Chrome/Edge 可执行文件。
