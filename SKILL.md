# 小红书内容运营大师（XHS Content Master）

把「方法论 + 出图 + 发布 + 合规」整合成一条可复用工作流，让任何账号都能稳定产出**不踩雷、能涨粉**的小红书图文笔记。

> 设计前提：**真人主导，AI 提效**。本技能帮真人把内容做得更快更好看，不做「全自动托管」。
> **铁律：任何发布/批量/互动动作，先读 `references/compliance-anti-ban.md` 过红线，再执行。**

---

## 0) 核心原则（每个任务都遵循）

1. **真人主导**：内容基于真实经验与判断，AI 负责组织、润色、排版、提效。
2. **稳定 > 爆发**：周更稳定、节奏自然，比偶尔刷屏更安全也更涨粉。
3. **封面是第一杠杆**：小红书信息流是白底明亮环境，封面要**大字、高对比、关键词高亮**才跳得出来。
4. **每篇可被关注**：每条笔记都要给用户「关注理由」，把收藏转化成关注。

---

## 1) 内容方法论模块

### 账号定位（先确认 4 个变量）
- 目标用户：年龄 / 场景 / 痛点
- 价值主张：每篇给用户什么（观点 / 情绪价值 / 实操）
- 差异化角度：同类账号不做、你做的
- 风格红线：语气、长度、不碰的底线

产出：人设关键词(3-5) + 内容支柱(3) + 固定句式(2-3) + 红线清单。

### 笔记五元组（每篇必备）
**标题**（≤20字，数字/疑问/冲突优先）· **开头钩子**（1-2句）· **正文**（观点→证据→延展，短句短段，每段 1-2 emoji）· **互动提问**（结尾1句引评论）· **标签**（5-8个：1大词+1人群词+1场景词）。

### 关注钩子（破解「收藏高不涨粉」）
每篇结尾明确告诉用户「关注你能持续得到什么」——这是把一次性流量沉淀为粉丝的关键。

> 选题、对标、爆款复刻的细化 SOP 见 `references/content-method.md`。

## 2) 出图模块（核心，自包含）

### 钩子封面（首选）
参数化模板 + headless Chrome 渲染，1080×1440，按内容类型分主题配色。

```bash
python scripts/render_cover.py <note.md> [-t blue|green|red|light] [-o out.png]
```

封面 Markdown 头部字段（YAML）：
```yaml
---
theme: blue            # blue=知识 / green=干货 / red=观点 / light=清新
tag: 左上角标签
eyebrow: （可选）标题上方引导小字
title: 主标题   # 支持 \n 换行，**关键词** 自动高亮
subtitle: 副标题  # 支持 **加粗**
chips: ["# 标签1", "# 标签2", "# 标签3"]
author: 你的昵称
author_sub: 一句话简介
---
```
> 主题与字段完整说明见 `references/cover-system.md`。

### 多卡片正文图（可选搭档）
需要把长 Markdown 一键切成「封面+多张正文卡片」、且想要更多排版皮肤时，
可搭配 [Auto-Redbook-Skills](https://github.com/comeonzhj/Auto-Redbook-Skills)（8 主题 + 4 分页模式）使用：
```bash
python render_xhs.py content.md -t <theme> -m auto-split
```
**组合建议**：封面用本技能（强钩子、配色分型）＋ 内页用多卡片渲染（快速成图）。

## 3) 发布模块

> ⚠️ 发布前**必过** `references/compliance-anti-ban.md` 红线检查。默认「仅自己可见」预览，确认无误再公开。

两种发布通道，按场景选：
- **浏览器手动/半自动**：在已登录小红书的浏览器中由真人确认后发布（最稳）。
- **脚本发布（可选）**：用 Cookie 方式的开源发布器（如 Auto-Redbook-Skills 的 `publish_xhs.py`），支持 `--dry-run`、定时、默认仅自己可见。Cookie 写入 `.env`，切勿提交到 Git。

发布合规要点（强制）：单账号每天 ≤1-2 篇、不刷屏；发布时间**随机化**（非整点机器式）；不在凌晨集中批量定时；**禁止**同设备多账号矩阵群发。

---

## 4) 合规与防封模块 ★（最高优先级）

**所有发布/互动动作执行前，先读并对照 `references/compliance-anti-ban.md`。**

一句话红线：**平台封的是「AI 全自动托管账号」，不是「用 AI 提效的真人」。**
保持「真人主导」四条命脉：① 内容基于真实经验、不造假 ② 发布前真人逐篇审核、不做全自动闭环 ③ 评论/私信真人回复、不开 AI 自动互动机器人 ④ 不搞同设备多账号矩阵、不批量同质化、AI 合成元素按规打标。

---

## 5) 标准工作流（端到端）

```
账号定位/选题(模块1) → 写笔记五元组(模块1) → 合规自查(模块4)
→ 封面=render_cover + 内页=多卡片(模块2) → 仅自己可见预览(模块3)
→ 真人确认 → 公开发布(模块3) → 评论区真人运营 → 复盘
```

每个任务结束：留下可检索的结论 / 证据 / 风险 / 下一步。
